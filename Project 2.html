SQL & KPI Analysis

Example Queries (SQLite‑style SQL)

The synthetic sheet is loaded into a SQLite table named calls. SQL is used to compute daily/weekly/monthly trends, queue performance, and days with unusual wait times.

Trends by Day / Week / Month

Track volume and average handle time at different time grains to support staffing and scheduling conversations. 

-- Daily call volume and average handle time
SELECT
    date(Date)                      AS Day,
    COUNT(*)                        AS Total_Calls,
    AVG(Handle_Time)                AS Avg_Handle_Time,
    AVG(Wait_Time)                  AS Avg_Wait_Time
FROM calls
GROUP BY date(Date)
ORDER BY Day;

-- Monthly trend for executive view
SELECT
    strftime('%Y-%m', Date)         AS Month,
    COUNT(*)                        AS Total_Calls,
    AVG(Handle_Time)                AS Avg_Handle_Time,
    AVG(Wait_Time)                  AS Avg_Wait_Time,
    AVG(CSAT_Score)                 AS Avg_CSAT
FROM calls
GROUP BY strftime('%Y-%m', Date)
ORDER BY Month;

Aggregate by Queue and Call Reason

Compare queues and reasons to identify where calls are most complex and where training or better self‑service content might help. 

-- Performance by queue
SELECT
    Queue,
    COUNT(*)                        AS Total_Calls,
    AVG(Handle_Time)                AS Avg_Handle_Time,
    AVG(Wait_Time)                  AS Avg_Wait_Time,
    AVG(CASE WHEN Resolution = 'Yes' THEN 1.0 ELSE 0 END) AS Resolution_Rate,
    AVG(CSAT_Score)                 AS Avg_CSAT
FROM calls
GROUP BY Queue
ORDER BY Avg_Handle_Time DESC;

-- Top call reasons by volume
SELECT
    Call_Reason,
    COUNT(*)                        AS Total_Calls,
    AVG(Handle_Time)                AS Avg_Handle_Time
FROM calls
GROUP BY Call_Reason
ORDER BY Total_Calls DESC;

Identify Anomalies in Wait Time

Flag days where the average wait time is significantly higher than normal, surfacing potential staffing issues, outages, or promotional spikes. 

-- Baseline average wait time across all days
WITH baseline AS (
    SELECT AVG(Avg_Wait) AS Global_Avg_Wait
    FROM (
        SELECT
            date(Date)        AS Day,
            AVG(Wait_Time)    AS Avg_Wait
        FROM calls
        GROUP BY date(Date)
    )
),
daily AS (
    SELECT
        date(Date)            AS Day,
        COUNT(*)              AS Total_Calls,
        AVG(Wait_Time)        AS Avg_Wait
    FROM calls
    GROUP BY date(Date)
)
SELECT
    d.Day,
    d.Total_Calls,
    d.Avg_Wait,
    b.Global_Avg_Wait,
    ROUND((d.Avg_Wait - b.Global_Avg_Wait) / b.Global_Avg_Wait * 100, 1)
        AS Deviation_Percent
FROM daily d
CROSS JOIN baseline b
WHERE d.Avg_Wait > b.Global_Avg_Wait * 1.35      -- >35% above normal
ORDER BY Deviation_Percent DESC;

Dashboard & Narrative

Dashboard Design

Built in Power BI Desktop (can also be implemented in Tableau Public or Looker Studio). The main page is structured for a team lead or operations manager who needs to quickly understand what happened today and how performance is trending.

Key Visuals

  • Total Calls by Day – column chart using daily counts to show demand and highlight spikes.
  • Average Handle Time by Queue – bar chart comparing AHT across queues to spot complex workflows.
  • Resolution Rate & CSAT Trend – line charts showing how quality and customer satisfaction move over time.
  • Wait Time Anomalies – table or scatter plot highlighting days with unusually high wait times.

Short Narrative (Business Language)

  • Identified that Queue B’s average handle time is ~35% higher than other queues, suggesting a need for either better call routing or targeted agent training.
  • Noticed a spike in calls from State X around billing cycle dates, indicating an opportunity to clarify billing communication and reduce repeat contacts.
  • Days with above‑normal wait times often line up with system outages and marketing campaigns, reinforcing the need for tighter coordination between IT, Marketing, and the call center.
  • Queues with higher resolution rates also show better CSAT scores, which supports focusing coaching on first‑contact resolution behaviors.

What This Project Proves

  • Understands a call‑center environment and the KPIs that matter.
  • Can find trends, patterns, and anomalies directly in the data.
  • Can translate numbers into clear, actionable language for departments.
  • Comfortable moving from raw data → SQL → dashboard → narrative.
